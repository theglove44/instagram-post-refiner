# Commit, Push, and Create PR

Commit all staged changes with an intelligent message, push to a feature branch, and create a pull request on GitHub for review.

## Process

1. Check git status and verify staged changes
2. Review the diff to understand what's being committed
3. Create a feature branch from main
4. Commit staged changes with a meaningful message
5. Push to the feature branch
6. Create a pull request on GitHub
7. Return the PR URL for review

## Expected output

- Confirmation of commit creation
- Feature branch name
- PR URL for review
